# 基本設計（UI/UX方針）

## 1. 目的
この設計は、非エンジニアや初学者でも「迷わず操作できる」ことを最優先にします。

## 2. 禁止事項
以下は、理解しづらさや誤操作につながるため採用しません。

- 過度なグラデーション、ガラスモーフィズム、意味のないアニメーション
- コントラスト不足テキスト、情報過密カード、装飾優先レイアウト
- CTA（行動ボタン）の乱立、アイコン依存でラベル不在の操作

## 3. 採用方針
- 余白と情報階層を優先した1カラム基調
- 視認性の高いフォント（`Noto Sans JP`, `Inter`, `system-ui` の順にフォールバック）
- フォームはラベル常時表示、エラー文は平易な日本語
- JavaScriptは入力チェックと非同期通信のみ（ゼロJS思想に沿う最小構成）

## 4. 画面設計ルール（全画面共通）
各画面の先頭に、以下のような「この画面で何ができるか」を1文で必ず表示します。

- 例: 「この画面では、文房具の在庫を登録できます。」

この1文を最初に置くことで、初学者でも目的を見失わずに操作できます。

## 5. 文章表現ルール
- 専門用語は避け、短い文で説明する
- エラー文は「原因」と「どう直すか」をセットで書く
- ボタン名は動詞で明確にする（例: 「保存する」「在庫を更新する」）
